POS System

Functions & Key Benefits
  • Master data ฐานข้อมูลเพื่อนำมาใช้งานในระบบโดยไม่ต้องบันทึกข้อมูลซ้ำ และเพื่อเป็นฐานข้อมูลเพื่อให้ระบบวิเคราะห์และประมวลผลข้อมูล สามารถอัพโหลด excel file เข้าในระบบเพื่อสร้าง master data ได้
  • Product master สามารถ mapping รหัสสินค้าของลูกค้ากับรหัสสินค้าที่ใช้ในระบบได้
  • Customer Quotation ใบเสนอราคาลูกค้า สามารถตั้งราคาขาย , ราคาต้นทุนสินค้า, ต้นทุนอื่นๆ เช่น ต้นทุนที่เกิดจากการซื้อหรือค่าใช้จ่ายที่เกี่ยวเนื่องกับการขายสินค้ารายการนั้น ๆ เพื่อให้ระบบคำนวณรายได้ กำไร/ขาดทุน โดยอัตโนมัติทันทีที่มีการเปิด S/O
  • Standard Quotation สามารถกำหนดราคาขายสำหรับลูกค้าทั่วๆ ไป ที่ไม่มีข้อตกลงเรื่องราคาที่แตกต่างไปจากลูกค้ารายอื่นๆ
  • Vendor Quotation ใบเสนอราคาของ vendor เช่นใบเสนอราคาของ ซัพพลายเออร์ ระบบสามารถคำนวณค่าใช้จ่ายที่ต้องจ่ายให้ vendor โดยอัตโนมัติ เมื่อมีการสร้าง Purchase order และ delivery order
  • P/O purchase order การออกใบสั่งซื้อระบบจะแสดงจำนวนสินค้าคงคลังของสินค้าแต่ละรายการที่ทำการออก P/O ให้ผู้สั่งซื้อเห็นจำนวนสินค้าคงเหลือและยอดค้างรับของสินค้าแต่ละรายการและความเคลื่อนไหวของสินค้า (cargo turnover)ในขณะที่ทำการสั่งซื้อ เพื่อป้องกันการสั่งซื้อมากเกินความสามารถในการระบายสินค้า สามารถแสดงรายการสินค้าที่ถึงจุดสั่งซื้อ (reorder point) เพื่อให้ฝ่ายจัดซื้อสามารถสั่งสินค้าได้ทัน ข้อมูลจะ link กับ good receipt เพื่อให้คนรับสินค้ารู้ข้อมูลของ PO ที่ซัพพลายเออร์จัดส่งว่าตรงตามใบสั่งซื้อหรือไม่
  • Good receipt การรับสินค้าเข้าคลังระบบจะอัพเดท stock แบบ real time และคำนวณปริมาณค้างรับของแต่ละ P/O user สามารถเรียกดูข้อมูลการรับสินค้าของ P/O นั้นๆ ได้ และระบบจะแนะนำตำแหน่งที่ควรจัดเก็บตามที่กำหนดไว้ใน master data
  • Warehouse Location สามารถกำหนดตำแหน่งการจัดเก็บสินค้าในคลังสินค้าได้ ว่าแต่ละ location ใช้สำหรับจัดเก็บสินค้าชนิดใด เพื่อที่ระบบจะสามารถแนะนำตำแหน่งการจัดเก็บที่เหมาะสมเมื่อมีการรับสินค้าเข้าคลัง (good receipt) และแนะนำตำแหน่งของสินค้าเมื่อมีการจำหน่ายสินค้า (sales order) เพื่อความสะดวกรวดเร็วในการหยิบสินค้า
  • Inventory control การควบคุม stock สามารถแสดง รายการสินค้า ปริมาณ หน่วย ต้นทุน ปริมาณค้างรับ ปริมาณค้างส่ง สถานที่จัดเก็บ(warehouse location) ซัพพลายเออร์ เป็นต้น และมีการตัด stock แบบ real time
  • S/O sales Order การขายสินค้า รองรับทั้งการขายเงินสดและเครดิต ระบบจะคำนวณค่าใช้จ่ายที่ลูกค้าจะต้องจ่ายจาการขายสินค้าแต่ละรายการตาม S/O นั้นๆ โดยระบบจะคำนวณราคาตาม quotation ของลูกค้าแต่ละราย สามารถให้ส่วนลดลูกค้าได้ทั้งแบบเปอร์เซ็นต์และเป็นจำนวนเงิน ระบบจะแสดงยอดสินค้าคงเหลือเพื่อให้ user ทราบว่ามีสินค้าพอที่จะจ่ายให้ลูกค้าตาม sales order หรือไม่ และจะเตือนเมื่อยอดคงเหลือต่ำกว่าจุดสั่งซื้อ (reorder point) เพื่อให้สั่งสินค้ามาทดแทน โดยระบบจะตัด stock ทันทีที่มีการขายสินค้า
  • Good issue การจ่ายสินค้าสามารถควบคุมการจ่ายสินค้าตามลำดับการจอง สามารถทยอยส่งสินค้าได้ โดยระบบจะคำนวณยอดค้างส่งให้โดยอัตโนมัติ user สามารถเรียกดูข้อมูลการจ่ายสินค้าของ S/O นั้นๆ ได้ สามารถสั่งพิมพ์ใบส่งของ delivery order ได้ทันที ระบบจะแนะนำตำแหน่งที่จัดเก็บสินค้าให้โอยอัตโนมัติตามที่ระบุไว้
  • Stock adjustment การปรับ stock เช่น กรณีที่ลูกค้าคืนสินค้า สินค้าชำรุด เป็นต้น ระบบจะอัพเดท stock แบบ real time
  • Unbilled order รายการเรียกเก็บเงินลูกค้าเครดิตที่ยังไม่ได้ออก invoiceหรือลูกค้าเงินสดที่ค้างจ่าย โดยสามารถเรียกข้อมูลเพื่อทำการออก invoice ได้หลากหลายรูปแบบ เช่น เรียกตามชื่อลูกค้า เรียกตามวันที่ปิด Order เรียกแบบรายสัปดาห์ รายเดือน ขึ้นอยู่กับความต้องการของ user และสามารถคลิกเพื่อออก invoice ได้จากหน้า unbilled order ได้ทันที โดยสามารถออก invoice รวมหลาย Order หรือแยกออก invoice ทีละ order ได้
  • Customer cash bill บิลเงินสดสำหรับลูกค้าเงินสด สามารถเลือกที่จะสั่งพิมพ์หรือไม่พิมพ์ใบเสร็จได้ตามต้องการ
  • Cash receipt confirmation การยืนยันการรับชำระเงินสดจากลูกค้า สามารกเรียกดูบิลเงินสดที่ออกในแต่วันหรือตามวันที่ต้องการเพื่อยืนยันการชำระเงินจริง
  • Customer invoiceการออกใบแจ้งหนี้สำหรับลูกค้าเครดิต สามารถกำหนดให้ระบบ ออก invoice และส่ง invoice ให้ลูกค้าโดยอัตโนมัติได้ (auto invoice) ตามเงื่อนไขและข้อตกลงของลูกค้าแต่ละราย เช่น ออก invoice รายสัปดาห์ รายเดือน หรือให้ออก invoice ทันทีที่ปิด order เป็นต้น หรือจะออก invoice แบบแมนนวลโดย user เป็นผู้ออกก็ได้
  • Vendor payment การทำจ่ายเงินให้ vendor ระบบสามารถคำนวณค่าใช้จ่ายที่ต้องจ่ายให้ซัพพลายเออร์แต่ละรายได้โดยอัตโนมัติ โดยคำนวณตามที่ระบุไว้ใน vendor quotation เช่น ทำจ่ายค่าวัสดุให้ซัพพลายเออร์ ทำจ่ายบริษัทขนส่ง
  • Cost sheet สรุปรายได้ ต้นทุน ค่าใช้จ่าย กำไร/ขาดทุน ของแต่ละ S/O ทั้งต้นทุนที่กำหนดไว้ตายตัวในฐานข้อมูล เช่น ต้นทุนราคาสินค้า ต้นทุนที่เกิดจากการซื้อสินค้า ต้นทุนค่าแรง เป็น
  • Inquiry สามารถค้นหาข้อมูลในระบบได้หลากหลายรูปแบบเช่น เรียกดูข้อมูลการขาย ข้อมูลสินค้า ข้อมูลลูกค้า ข้อมูลคลัง ข้อมูล invoice เป็นต้น ด้วยระบบ advance search
  • Report รายงาน สามารถส่งรูปแบบรายงานที่ต้องการให้ทางเราสร้างรายงานดังกล่าวในระบบได้หรือจะให้ทางเราเป็นผู้ออกแบบให้ก็ได้ ระบบสามารถแสดงข้อมูลแบบในรูปแบบของกราฟได้ทุกชนิด
  • Barcode รองรับระบบบาร์โค้ด
  • Auto back up ระบบจะทำการ back up ข้อมูลให้โดยอัตโนมัติ
